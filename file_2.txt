Задание 5. Git и Yandex Tracker
Ответом на задание должны быть две ссылки: на ваш репозиторий на Github и на ваш сервис в Yandex Tracker
В пунктах по Git под созданием коммита подразумевается не только его добавление в локальную копию репозитория (git commit), но и сразу его отправка на сервер (git push)
Задания двух блоков оцениваются поровну, т.е. 3 балла за Git и 3 балла за Yandex Tracker

Git:
Зарегистрируйтесь на github.com, если вы этого ещё не сделали. Сделайте проект на любом языке программирования, который решает какую-то простую задачу (например, оптимально решает задачу коммивояжёра). Поместите свой проект в публичный репозиторий.
Настройте репозиторий так, чтобы .bin-файлы автоматически игнорировались. Попытайтесь добавить в репозиторий произвольный .bin-файл, убедитесь, что этого не произошло.
Отдельным коммитом добавьте в репозиторий пустой txt-файл. Каждый член вашей команды должен сделать форк репозитория с пустым файлом, дописать в файл свою фамилию, после чего сделать пул-реквест в основной репозиторий. Владелец репозитория должен принять все пул-реквесты.
Создайте новую ветку от коммита с пустым txt-файлом, в новом коммите запишите туда произвольный текст. Затем откатите последние правки и сделайте слияние новой ветки с дефолтной.
Создайте ещё одну ветку, в ней в трёх разных коммитах добавьте три произвольных текстовых файла. Перенесите в дефолтную ветку только тот файл, который был добавлен вторым.
Yandex Tracker:
Откройте свой сервис на сайте https://cloud.yandex.ru/services/tracker (может потребоваться аккаунт в Яндексе), добавьте каждого члена команды в сервис, а также преподавателя (моя почта - mozdorov@yandex.ru).
Создайте очередь с произвольным названием. Назначьте владельцем другого члена команды (не того, кто создавал сервис). Настройте права доступа так, чтобы редактировать задачи могли только их авторы. Настройте рассылку уведомлений по всем задачам для всех участников команды и преподавателя.
Каждый участник команды должен создать по одной задаче в созданной очереди. Автором нужно назначить себя, исполнителем — любого другого члена команды (сделайте так, чтобы каждый был исполнителем какой-то задачи), наблюдателем — преподавателя. Если вы в команде один/одна, то назначьте исполнителем тоже себя. Остальные поля можно заполнить произвольно.
Создайте спринт продолжительностью 4 недели, датой начала выберите дату публикации этого задания. Поместите все созданные задачи в данный спринт и запустите его.
Каждый участник команды должен провести задачу, где он назначен исполнителем: нужно последовательно изменить статусы Открыт -> В работе -> Закрыт, причём в статусе "В работе" задача должна пробыть не менее 5 минут. После этого завершите спринт.
